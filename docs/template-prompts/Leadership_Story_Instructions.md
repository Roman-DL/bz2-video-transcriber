# Инструкция: Анализ историй лидеров Herbalife

## Назначение

**Конспект лидерской истории** — структурированный документ для:
- Быстрого понимания пути лидера
- Сравнения с другими историями
- Поиска паттернов (стагнация, прорыв, методы работы)
- RAG-запросов («кто прошёл через X», «примеры Y»)

**Лонгрид** — опционально, для длинных историй (30+ минут), где важно сохранить детали и голос автора.

---

## Принципы

1. **Фокус на уникальном** — не пересказ, а что ОТЛИЧАЕТ эту историю
2. **Конкретика** — цифры, даты, факты. Не «много», а «53 СВ за месяц»
3. **Гибкость** — заполнять только релевантные блоки
4. **Не придумывать** — если данных нет, пропустить поле или блок
5. **Исправлять ошибки** — использовать внешний глоссарий (glossary.yaml)
6. **Максимум деталей** — каждая фишка = точка входа для поиска

## Работа со слайдами

### Что обычно есть на слайдах историй
- **Хронология** — даты статусов, ключевые периоды
- **Цифры** — объёмы, количество СВ, GET-линий
- **Методики** — схемы работы, инструменты
- **Фото** — визуальные маркеры периодов

### Что извлекать
- Точные даты статусов (если не озвучены)
- Цифры квалификаций
- Названия инструментов и методик
- Структуру бизнеса (клуб/онлайн/гибрид)

### Как интегрировать
Данные со слайдов вносить в соответствующие поля шаблона:
- Даты → блок «Путь в бизнес» и «Рост и вызовы»
- Цифры → блок «Ключ к статусу» и «Итоги»
- Методики → блок «Как устроен бизнес»

## Метаданные

```yaml
---
type: leadership-story
names: "Имя1 и Имя2"
current_status: "President's Team"
event: "Название мероприятия"
date: YYYY-MM-DD

# Характеристики пути
time_in_business: "N лет"
time_to_status: "N лет"
speed: средне                  # быстро | средне | долго | очень долго

# Формат бизнеса
business_format: гибрид        # клуб | онлайн | гибрид

# Особенности истории
is_family: false
had_stagnation: false
stagnation_years: 0
had_restart: false
key_pattern: ""
mentor: ""

# Доступ и публикация
access_level: consultant       # consultant | leader | personal
publish_gdocs: false

# Для поиска и связей
tags: []
related: []
---
```

### Значения speed

| Значение | Время до текущего статуса |
|----------|---------------------------|
| быстро | менее 3 лет |
| средне | 3–7 лет |
| долго | 7–15 лет |
| очень долго | более 15 лет |

### Значения business_format

| Значение | Описание |
|----------|----------|
| клуб | Помещение для встреч, офлайн-мероприятия |
| онлайн | Zoom, марафоны, удалённая работа |
| гибрид | Совмещение клуба и онлайна |

## Структура шаблона

Шаблон содержит **8 блоков**. Заполнять по наличию информации:

| # | Блок | Когда заполнять |
|---|------|-----------------|
| 1 | **Кто они** | Всегда — базовая информация |
| 2 | **Путь в бизнес** | Всегда — как пришли, первые шаги |
| 3 | **Рост и вызовы** | Если есть хронология, стагнации |
| 4 | **Ключ к статусу** | Если описан прорыв/квалификация |
| 5 | **Как устроен бизнес** | Если описана система работы |
| 6 | **Принципы и советы** | Если есть философия, правила |
| 7 | **Итоги** | Всегда — сводка и классификация |
| 8 | **Заметки аналитика** | Всегда — для RAG и сравнения |

**Правило:** Пустые поля и подблоки удалять из финального документа.

## Правила извлечения

### Что включать
- Конкретные цифры и даты
- Уникальные методики и фишки (5-7 штук)
- Переломные моменты и решения
- Причины стагнаций и способы выхода
- Прямые цитаты (3-5 ярких)
- Контекст решений («почему так сделали»)
- Правила и ритуалы команды
- Авторские инструменты и скрипты

### Что НЕ включать
- Общие фразы без конкретики
- Благодарности и приветствия
- Повторы одной мысли
- Информацию, не относящуюся к бизнес-пути

### Уникальные фишки (5-7 штук)

Извлекать ВСЕ уникальные практики, даже мелкие:
- Авторские методики и инструменты
- Правила команды (писаные и неписаные)
- Необычные подходы к работе с клиентами/командой
- Фразы, скрипты, которые используют
- Ритуалы и традиции команды
- Системы мотивации и контроля
- Принципы позиционирования

**Каждая фишка = точка входа для RAG-поиска.**

### Особые случаи

**Короткая история (10-15 минут):**
- Заполнять только блоки с информацией
- Блоки 4, 5 могут быть пустыми — удалить

**Фокус на хронологии:**
- Подробно заполнить блоки 2, 3
- Использовать таблицу этапов

**Фокус на уроках:**
- Подробно заполнить блоки 5, 6
- Больше цитат и принципов

## Формат вывода

- YAML-метаданные в начале

- 3-5 прямых цитат
- Главный инсайт — 1 предложение в начале

**Имя файла:** `Фамилия (Имя1 и Имя2).md` или `Фамилия (Имя).md`

## Блок «Для кого эта история»

Отмечать **с конкретным объяснением почему** — критично для поиска:

```markdown
- [x] **Новички** — пример: 10 лет стагнации не приговор, если есть решение
- [x] **Супервайзеры** — правило «никакого следующего месяца» применимо сразу
- [ ] **Работающие онлайн** — история про офлайн, принципы универсальны
```

**Правила:**
- Объяснение должно быть конкретным
- Не писать «полезно для всех» — это бессмысленно для поиска
- Если не релевантно — не отмечать галочкой, но можно добавить комментарий

## Блок «Заметки аналитика»

Заполнять ВСЕГДА. Содержит:

### Уникальные паттерны
- Что выделяет эту историю среди других
- Редкие кейсы и необычные решения
- Рекордные или примечательные цифры

### Связанные истории
Заполняется позже при наличии базы:
- **Похожие** — по какому признаку
- **Контрастные** — в чём отличие
- **Упомянутые лидеры** — роль в истории

### Открытые вопросы
- Что осталось неясным из транскрипта
- Что стоит уточнить при возможности

## Сравнение историй

### Паттерны для тегирования

При анализе отмечать паттерны в tags:

**По пути:**
- `быстрый-старт` — СВ за 1-2 месяца
- `медленный-старт` — СВ через 6+ месяцев
- `стагнация-N-лет` — период без роста
- `перезапуск` — начали заново
- `переезд` — сменили город/страну

**По формату:**
- `трансформация-в-онлайн` — перешли с клуба
- `открытие-клуба` — запустили помещение
- `гибрид-с-нуля` — сразу оба формата

**По команде:**
- `семейная-пара` — работают вместе
- `большая-первая-линия` — 10+ в первой линии
- `глубина-важнее-ширины` — фокус на развитие вглубь
- `инициатива-команды` — рост по инициативе нижестоящих

**По прорыву:**
- `мероприятие-как-триггер` — рост после события
- `наставник-как-триггер` — влияние спонсора
- `кризис-как-триггер` — внешние обстоятельства

**По методам:**
- `днк-команды` — прописанные принципы и правила
- `открытые-цифры` — показывают результаты друг другу
- `дресс-код` — требования к внешнему виду

## Лонгрид для историй (опционально)

Для длинных историй (30+ минут) можно создать лонгрид — полное изложение от первого лица.

**Когда делать:**
- История содержит много деталей и нюансов
- Важен авторский стиль изложения
- Планируется использовать для обучения

**Порядок:**
1. Сначала лонгрид (по инструкции для лонгридов)
2. Затем конспект на основе лонгрида

**Файлы:**
- Лонгрид: `Фамилия (Имя1 и Имя2).md`
- Конспект: `Фамилия (Имя1 и Имя2) — конспект.md`
