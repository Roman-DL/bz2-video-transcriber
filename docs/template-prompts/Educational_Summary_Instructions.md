# Инструкция: Конспект обучающей темы

## Назначение конспекта

**Конспект — это навигационный документ для тех, кто уже слушал тему.**

Конспект помогает:
- Быстро вспомнить ключевые моменты
- Найти конкретный инструмент или принцип
- Освежить в памяти перед применением

Конспект НЕ заменяет:
- Прослушивание темы (для первичного понимания)
- Лонгрид (для глубокого изучения без записи)

---

## Принципы

1. **Структура важнее полноты** — выделить скелет темы, а не пересказать всё
2. **Конкретика** — цифры, названия, точные формулировки
3. **Цитаты для ключевых моментов** — 2-4 прямые цитаты спикера
4. **Не придумывать** — если данных нет, пропустить блок
5. **Выбирать блоки по типу темы** — не все блоки обязательны
6. **Исправлять ошибки транскрибации** — использовать глоссарий

---

## Метаданные

```yaml
---
type: конспект
speaker: "Имя Фамилия"
speaker_status: "Статус"
event: "Название мероприятия"
date: YYYY-MM-DD

topic_area:
  - область1
  - область2

access_level: consultant  # consultant | leader | personal
publish_gdocs: false

tags: []
related:
  - "[[Фамилия — Название темы]]"  # ссылка на лонгрид
---
```

### Допустимые значения topic_area

| Значение | Описание |
|----------|----------|
| `продажи` | Работа с клиентами, первички, продажи программ |
| `спонсорство` | Работа с командой, планирование, 3D-чаты |
| `лидерство` | Построение организации, первое поколение |
| `мотивация` | Философия, мышление, преодоление кризисов |
| `инструменты` | Конкретные методики (дом-магазин, солнышко) |
| `маркетинг-план` | Статусы, квалификации, роялти |

## Вариативные блоки

Шаблон содержит два набора блоков. **Выбирай по содержанию темы:**

### Методическая тема
Используй, если спикер даёт конкретные инструменты и пошаговые инструкции.

| Блок | Когда использовать |
|------|-------------------|
| **Инструменты и методы** | Есть названные инструменты (дом-магазин, 3D-чат, солнышко) |
| **Алгоритм действий** | Есть чёткая последовательность шагов |
| **Типичные ошибки** | Спикер говорит, как делать НЕ надо |

### Философская тема
Используй, если спикер говорит о мышлении, принципах, мотивации.

| Блок | Когда использовать |
|------|-------------------|
| **Ключевые принципы** | Есть фундаментальные идеи, убеждения |
| **Ментальные ловушки** | Спикер разбирает заблуждения, иллюзии |

### Смешанная тема
Используй блоки из обоих наборов по необходимости.

---

## Общие блоки (всегда)

| Блок | Описание |
|------|----------|
| **Суть темы** | Главная идея в 1-2 предложениях |
| **Ключевые концепции** | Определения, термины, важные различия |
| **Истории и примеры** | Конкретные кейсы из выступления |
| **Ключевые цитаты** | 2-4 прямые цитаты спикера |
| **Главный инсайт** | Одна мысль, которую унести |
| **Вопросы и ответы** | Если были Q&A после темы |
| **Применение** | 2-3 действия для внедрения |

## Правила извлечения

### Что включать
- Конкретные инструменты с названиями
- Пошаговые алгоритмы (если есть)
- Принципы и убеждения спикера
- Примеры с цифрами и деталями
- Яркие цитаты
- Предостережения и антипримеры

### Что НЕ включать
- Общие фразы без конкретики
- Повторы одной мысли
- Отступления от темы
- Технические моменты (аплодисменты, паузы)
- Слова ведущих до/после темы

### Работа с неполными данными
- Инструмент упомянут, но не объяснён → указать название + "детали не раскрыты"
- Нет чёткого алгоритма → пропустить блок "Алгоритм действий"
- Нет примеров → пропустить блок "Истории и примеры"
- Нет Q&A → удалить блок "Вопросы и ответы"

---

## Формат вывода

- YAML-метаданные в начале
- Вариативные блоки по типу темы
- Общие блоки (пропускать пустые)
- Каллауты Obsidian: `[!abstract]`, `[!info]`, `[!tip]`, `[!example]`, `[!warning]`, `[!quote]`, `[!success]`, `[!todo]`
- 2-4 прямые цитаты
- Главный инсайт — 1 предложение

**Объём:** 2-4 страницы

**Имя файла:** `Фамилия — Название темы (конспект).md`

---

## Связь с лонгридом

| Аспект | Конспект | Лонгрид |
|--------|----------|---------|
| Назначение | Вспомнить | Понять |
| Аудитория | Был на теме | Не был на теме |
| Структура | По типам информации | По логике автора |
| Стиль | Тезисы, списки | Связное изложение |
| Объём | 2-4 страницы | 5-10 страниц |
| RAG | Навигация | Основной источник |
