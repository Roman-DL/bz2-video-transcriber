# ADR-019: Opus default для генерации конспекта

**Статус:** Принято
**Дата:** 2026-02-21

## Контекст

Конспект — навигационный документ для тех, кто уже слушал тему. Хранится в Obsidian как личная заметка. Используется для быстрого «освежения памяти» перед обращением к лонгриду или RAG. Должен быть компактным, структурированным и легко сканируемым.

С v0.29 (ADR-007) конспект генерируется Claude. Default модель — Sonnet. Проведено A/B сравнение Sonnet 4.6 vs Opus 4.6 на реальном транскрипте (~10K слов, тема «Спонсорство как искусство»).

### Результаты сравнения

| Критерий | Sonnet 4.6 | Opus 4.6 |
|----------|-----------|----------|
| Полнота тезисов (7 ключевых) | 5/7 | **6/7** |
| Точность передачи смысла | 3/5 | **5/5** |
| Компактность | 3/5 | **4/5** |
| Сканируемость | 3/5 | **5/5** |
| Структура | 4/5 | 4/5 |
| Actionable items | 4/5 | **5/5** |
| **Итого** | **22/32** | **29/32** |

**Sonnet** пропустил фокусную группу (центральный инструмент выступления), разбил цитаты на 7 фрагментов вместо цельных мыслей, не выделил ряд «забираемых» инструментов в действиях.

**Opus** покрыл больше тезисов, выбрал 4 цельные цитаты, включил фокусную группу как отдельное действие, лучше приоритизировал что главное vs побочное.

### Ключевой аргумент: diversity of topics

Конспект обрабатывает очень разные темы. Проблемы Sonnet — не промпт-баги, а ограничения модели:

- **Пропуск центрального упражнения** — Sonnet не смог верно приоритизировать. Промпт «не пропускай упражнения» поможет на этой теме, но на следующей Sonnet может пропустить что-то другое
- **Фрагментация цитат** — неспособность оценить границы «завершённой мысли». Opus интуитивно понимает, Sonnet требует правил на каждый edge case
- **Философские/смешанные темы** — требуют от модели глубокого понимания «что важно для reader'а». Более мощная модель справляется стабильнее

### Стоимость

Конспект — компактный документ (~500 слов вывода, ~10K слов ввода). Стоимость на Opus — копейки, разница с Sonnet несущественна.

## Решение

1. **Opus — default модель для конспекта** (env: `SUMMARY_MODEL=claude-opus-4-6`)
2. **Промпты обновлены** — убраны жёсткие числовые лимиты, добавлены качественные критерии:
   - Цитаты: «завершённая мысль, понятная без контекста» вместо «2-4 цитаты»
   - Actions: «столько, сколько реально даёт тема» вместо «2-3»
   - Новые правила: извлекать формулы/фреймворки, интерактивные упражнения
   - Развёрнутое описание смешанной темы (самый частый тип)

Sonnet остаётся доступной альтернативой через UI (выбор модели на шаге).

## Последствия

### Положительные
- Более надёжная приоритизация ключевых идей на разнообразии тем
- Лучше качество цитат — цельные мысли вместо фрагментов
- Стабильнее результат на философских и смешанных темах
- Промпты без жёстких лимитов — модель адаптируется к богатству темы

### Отрицательные
- Чуть дороже (разница несущественна для конспекта)
- Медленнее генерация (~30-40 сек vs ~15-20 сек)

## Альтернативы

### 1. Остаться на Sonnet + улучшить промпты
- ✗ Подход «чинить промптами» хрупкий — каждый тип темы выявляет новые edge cases
- ✗ Приоритизация — проблема понимания модели, не формата промптов
- ✓ Дешевле и быстрее

### 2. Sonnet с жёсткими ограничениями
- ✗ Жёсткие лимиты (2-4 цитаты, 2-3 действия) — костыль для слабой модели
- ✗ На богатых темах теряется ценный контент
- ✓ Предсказуемый формат
