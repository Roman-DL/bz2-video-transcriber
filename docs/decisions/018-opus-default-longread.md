# ADR-018: Opus default для генерации лонгрида

**Статус:** Принято
**Дата:** 2026-02-21

## Контекст

Лонгрид — ключевой артефакт pipeline: отредактированная версия выступления для тех, кто его не слушал. Качество лонгрида напрямую влияет на полезность RAG базы знаний.

С v0.29 (ADR-007) система использует Claude. Default модель для лонгрида — Sonnet. Проведено A/B сравнение Sonnet 4.6 vs Opus 4.6 на реальном транскрипте (~10K слов, тема «Спонсорство как искусство»).

### Результаты сравнения

| Критерий | Sonnet 4.6 | Opus 4.6 |
|----------|-----------|----------|
| Полнота покрытия (7 тезисов) | 5/7 | **7/7** |
| Галлюцинации | Нет | Нет |
| Исправление ошибок Whisper | Лучше в 1 случае | Хуже в 1 случае |
| Структура для RAG | 12 разделов, 1 >600w | **21 раздел, все <600w** |
| Голос спикера | Местами суховат | **Больше живых цитат** |
| Равномерность | 36% перекос | **28% перекос** |
| Стоимость | ~$0.17 | ~$0.46 |

**Sonnet** упустил 2 из 7 ключевых примеров и создал раздел на 626 слов (превышает лимит чанкера).
**Opus** включил все примеры, лучше сохранил голос спикера, но создал 21 раздел вместо оптимальных 8-15 (избыточное дробление).

### Стоимость

- ~$0.46 за лонгрид (Opus) vs ~$0.17 (Sonnet)
- ~10 тем/месяц → $4.60 vs $1.70 — разница несущественна
- Общая обработка темы ~$0.90 — лонгрид составляет ~50%

## Решение

1. **Opus — default модель для лонгрида** (env: `LONGREAD_MODEL=claude-opus-4-6`)
2. **Промпты дополнены** для компенсации избыточного дробления Opus:
   - Рекомендация 8-15 разделов для выступления 30-90 минут
   - Требование равномерной детализации от начала до конца

Sonnet остаётся доступной альтернативой через UI (выбор модели на шаге).

## Последствия

### Положительные
- Полнее покрытие материала — не теряются примеры и истории
- Лучше сохранение голоса спикера
- Лучшая готовность к сложным случаям (2+ спикера, длинные темы)
- Промпт-ограничения решают проблему дробления

### Отрицательные
- x2.7 стоимость на этап лонгрида (~$0.29 разницы на тему)
- Медленнее генерация (~40-60 сек vs ~20-30 сек)
- Дороже при отладке промптов (много прогонов на реальных данных)

## Альтернативы

### 1. Остаться на Sonnet + улучшить промпты
- ✗ Пропуск примеров — проблема понимания, не формата; промптами не надёжно решается
- ✗ Для сложных тем (2 спикера) разрыв в качестве увеличится
- ✓ Дешевле при отладке

### 2. Opus для «премиум», Sonnet для «стандарт»
- ✗ Усложняет UX — пользователь не может оценить разницу заранее
- ✓ Гибкость по стоимости
- → Фактически доступно через выбор модели в UI
