# Статус загрузки материалов в базу знаний

**Дата:** 2026-03-18
**Статус:** Done (v0.82.0)

## Проблема

Материалы из архива транскрайбера загружаются в базу знаний (bz2-bot) вручную. Сейчас нет способа увидеть, какие материалы уже загружены, а какие ещё нет. Это приводит к необходимости помнить статус каждого материала.

## Решение

Файл-маркер `.published` в папке материала. Наличие файла = материал загружен в БЗ.

```
archive/2025/ПШ/05.05 НП. Работа с возражениями (Петрова Мария)/
├── transcript_chunks.json
├── ...
└── .published    ← пустой файл-маркер
```

### Почему файл-маркер

- Нулевая связность между проектами — каждый работает с файловой системой независимо
- Одна операция на проверку (`os.path.exists`), создание (`touch`), удаление (`unlink`)
- Нет парсинга, нет формата, нет версионирования
- Работает офлайн

## Изменения: bz2-video-transcribe

### Backend

#### Модель `ArchiveItem`

Добавить поле `published: bool = False` в `ArchiveItem` (`backend/app/models/schemas.py`):

```python
class ArchiveItem(CamelCaseModel):
    title: str
    speaker: str
    event_type: str
    topic_folder: str
    published: bool = False  # загружен в БЗ
```

#### Эндпоинт `GET /api/archive`

При сканировании папок материалов проверять наличие `.published`:

```python
published = (topic_path / ".published").exists()
item = ArchiveItem(..., published=published)
```

Поле передаётся во фронтенд как `published: true/false` (camelCase в JSON).

Добавить `published_total: int` в `ArchiveResponse` — количество опубликованных материалов. Считается при сканировании вместе с `total`.

#### Эндпоинт `DELETE /api/archive/published` — снятие метки

Параметры: `year`, `event_group`, `topic_folder` (query).

Логика:
1. Валидация пути: построить путь к папке материала, проверить что он внутри `settings.archive_dir` (`resolved.relative_to(archive_dir)`)
2. Удалить `.published` если существует
3. Вернуть 200 OK (идемпотентный — 200 даже если файла не было)

#### Эндпоинт `PUT /api/archive/published` — установка метки

Те же параметры. Создаёт `.published` в папке материала. Для ручной пометки из UI.
Идемпотентный — `touch()` не падает если файл уже есть.
Валидация пути аналогична DELETE.

### Frontend

#### Дерево архива (`ArchiveCatalog.tsx`)

Рядом с опубликованными материалами — компактный зелёный бейдж "В БЗ":

```
📁 2025 (8)
  📁 ПШ (8)
    📄 03.10 НП. Закрытие сделки на ЭО и ПО (Товстая Наталья)  ✅ В БЗ
    📄 04.28 НП. Навыки консультанта по питанию (Романова Ольга)  ✅ В БЗ
    📄 05.05 НП. Работа с возражениями (Петрова Мария)
```

- Опубликованные — бейдж "В БЗ" (зелёный, ненавязчивый)
- Неопубликованные — без бейджа

#### Счётчик в заголовке

Рядом с общим количеством видео добавить количество опубликованных:

```
Архив  18 видео · 10 в БЗ
```

#### Переключатель статуса БЗ (`ArchiveResultsModal`)

В модалке результатов — универсальная кнопка-переключатель статуса публикации:

- Если материал **опубликован** — кнопка "Снять метку БЗ" → вызов `DELETE /api/archive/published`
- Если материал **не опубликован** — кнопка "Отметить как загружен" → вызов `PUT /api/archive/published`

После действия — инвалидация кеша архива (React Query refetch), бейдж обновляется в дереве.

## Миграция: начальная разметка существующих материалов

Все материалы типа ПШ уже загружены в базу знаний. При деплое (или одноразовым скриптом) нужно создать `.published` во всех папках групп ПШ:

```bash
# Создать .published во всех папках материалов внутри групп ПШ
find /mnt/main/work/bz2/video/archive/*/ПШ/ -mindepth 1 -maxdepth 1 -type d -exec touch {}/.published \;
```

Скрипт добавить в `scripts/` для воспроизводимости (например `scripts/mark_published.sh`).

## Изменения: bz2-bot (отдельная реализация)

### Адаптация под 3-уровневую структуру архива

Текущий `browse_archive()` (`src/admin/services/archive.py`) — generic файловый браузер с пошаговой навигацией через `relative_path`. Архив транскрайбера теперь использует 3-уровневую структуру:

```
archive/{year}/{event_group}/{topic_folder}/
```

Нужно заменить пошаговую навигацию на структурированное отображение дерева `год → группа → материалы`, аналогично тому, как это реализовано в транскрайбере (`GET /api/archive` → `ArchiveResponse`). Это позволит видеть весь список материалов сразу.

### Фильтрация по `.published`

При сканировании leaf-папок (материалов) — пропускать те, где есть `.published`:

```python
if (topic_path / ".published").exists():
    continue  # уже загружен в БЗ
```

Опциональный параметр `show_published: bool = False` в API для возможности увидеть все материалы.

### Создание маркера при загрузке

При успешном вызове `load_from_folder()` — создавать `.published` в папке материала:

```python
(topic_path / ".published").touch()
```

### Удаление маркера при удалении материала (опционально, на будущее)

При удалении материала из БЗ — удалять `.published` из архивной папки. Пока реализуется ручное снятие метки через UI транскрайбера.
