# Поддержка иностранных транскриптов с переводом на русский

**Дата:** 2026-03-26
**Статус:** Done (v0.83)

## Контекст

Появились кейсы обработки англоязычных (и мультиязычных) выступлений. Транскрипты подготавливаются вручную как `.md` файлы. Особенности:

- Транскрипт может содержать несколько языков (EN + ES и др.) — например, когда ведущий приглашает испаноговорящего гостя
- Спикеры обозначены как `Speaker N` (имена определяются из контекста)
- Многоспикерные выступления бывают и на русском языке
- Конечные пользователи контента — русскоговорящие

## Решение: Translate-in-Longread

Перевод встраивается в этап Longread/Story. Новый этап pipeline не добавляется.

### Почему не отдельный Translate stage

- Claude Sonnet 4.6 нативно работает с мультиязычным вводом
- Один LLM-вызов вместо двух (экономия времени и токенов)
- Коррекция ошибок транскрибации происходит естественно при генерации
- Минимум изменений в архитектуре pipeline

## Архитектура

### Pipeline flow

**Иностранный транскрипт:**
```
MD файл → Parse (язык=foreign) → Transcribe (pass-through) → Clean (SKIP) → Longread/Story (перевод+генерация) → Chunk → Save
```

**Русский транскрипт (без изменений):**
```
MD файл → Parse (язык=ru) → Transcribe (pass-through) → Clean (глоссарий) → Longread/Story → Chunk → Save
```

## Компоненты

### 1. Определение языка (Parse stage)

- Автодетекция по содержимому `.md` файла
- Берём первые ~500 символов текста (без заголовков и меток спикеров)
- Если >50% символов кириллица → `"ru"`, иначе → `"foreign"`
- Конкретный язык не определяем — важно только "русский или нет"
- Новое поле в `VideoMetadata`: `language: str = "ru"`

### 2. Условный пропуск Clean stage

- Если `metadata.language == "foreign"` → Clean пропускается
- Возвращаем `CleanedTranscript` с оригинальным текстом (pass-through)
- Причина: глоссарий содержит русскоязычные `whisper_errors` — для иностранного текста подстановка бессмысленна
- Коррекция ошибок транскрибации произойдёт на этапе Longread/Story

### 3. Модификация промптов Longread/Story

Динамический блок инструкций, инжектируемый при `language == "foreign"` (аналогично `speaker_context`):

```
## Иностранный транскрипт

Транскрипт содержит речь на иностранном языке (возможно несколько языков).
- Генерируй текст ПОЛНОСТЬЮ на русском языке
- Используй глоссарий как справочник терминов — применяй canonical формы
- Имена собственные транслитерируй (John Smith → Джон Смит)
- Если в транскрипте есть ошибки транскрибации — исправляй по контексту
```

Реализация: функция `build_language_context(metadata)` рядом с существующей `build_speaker_context()`. Глоссарий передаётся как обычно, но с ролью "справочник терминов".

Отдельные промпт-файлы для foreign НЕ создаются — один промпт, блок добавляется динамически.

### 4. Обработка спикеров в иностранных транскриптах

- Парсинг меток `Speaker N` уже работает в `speaker_utils.py` — изменения не нужны
- Сценарии определяются автоматически: 2 спикера → `co_speakers`, 3+ → `lineup`
- В промпт добавляется инструкция: определить имена спикеров из контекста (представление ведущим, самопрезентация) и использовать их вместо меток
- Транслитерация имён на русский — часть блока инструкций из п.3

### Сценарии спикеров

| Сценарий | Поведение |
|----------|-----------|
| 2 со-спикера | Единый текст. В story — атрибуция историй ("история Джона") |
| 3+ линейка | Секции по спикерам с подзаголовками |

## Изменяемые файлы

| Компонент | Файл | Изменение |
|-----------|------|-----------|
| Модель | `backend/app/models/schemas.py` | Поле `language: str = "ru"` в `VideoMetadata` |
| Parse | `backend/app/services/stages/parse_stage.py` | Автодетекция языка по кириллице |
| Clean | `backend/app/services/stages/clean_stage.py` | Pass-through при `language == "foreign"` |
| Language context | `backend/app/utils/` | Функция `build_language_context()` |
| Longread stage | `backend/app/services/stages/longread_stage.py` | Инжекция `language_context` в промпт |
| Story stage | `backend/app/services/stages/story_stage.py` | Инжекция `language_context` в промпт |

## Что НЕ меняется

- `speaker_utils.py` — парсинг уже работает
- Chunk, Summary, Save — работают с русским текстом из Longread/Story
- Frontend — язык определяется автоматически
- Конфигурация моделей — Sonnet 4.6 справляется
- Глоссарий — используется как есть
