# Унификация структуры архивных папок

**Дата:** 2026-03-18
**Статус:** Done (v0.82.0)

## Проблема

Текущая 4-уровневая структура архива неудобна для навигации:
- **Regular (ПШ):** `archive/{year}/{event_type}/{MM.DD}/{topic}/` — папка `MM.DD` содержит ровно одну запись, приходится раскрывать каждую
- **Offsite (ФСТ):** `archive/{year}/Выездные/{event_type}/{topic}/` — нет даты, непонятна хронология; лишний уровень «Выездные»

Дополнительно: детекция leadership (`title == "История"`) не покрывает кейсы с квалификацией (`История AWT`, `История GET`) и конфликтует с образовательным контентом (`История Herbalife и истоки`).

## Решение

### 1. Унифицированная 3-уровневая структура

```
archive/{year}/{event_group}/{topic_folder}/
```

| Категория | event_group | topic_folder |
|-----------|-------------|--------------|
| **Regular** | `{event_type}` | `{MM.DD} {stream}. {title} ({speaker})` (с потоком) |
| **Regular** | `{event_type}` | `{MM.DD} {title} ({speaker})` (без потока) |
| **Offsite** | `{MM} {event_type}` | `{title} ({speaker})` |

### 2. Примеры

**Regular:**
```
2025/ПШ/08.04 НП. Контент (Пепелина Инга)/
2025/ПШ/08.04 SV. История AWT (Прохорова Светлана)/
2025/ВВК/12.22 Тестовая запись (Тест)/
```

**Offsite:**
```
2026/02 ФСТ/В Herbalife мечты сбываются (Семенова Антонина и Игорь)/
2026/02 ФСТ/История Herbalife и истоки (Руцман Ида)/
2026/01 Форум TABTeam/История PT (Антоновы Дмитрий и Юлия)/
```

### 3. Leadership detection: `#История`

- **Было:** `title == "История"` — точное совпадение
- **Стало:** `title.startswith("#История")` — маркер `#` в имени файла

**`#` — маркер только на входе.** Парсер потребляет `#` и убирает его из `metadata.title`:
```
Входной файл:  2026.03.16 ПШ.НП. #История AWT (Прохорова Светлана).mp4
Parser output:  title="История AWT", content_type=LEADERSHIP
Archive path:   2025/ПШ/03.16 НП. История AWT (Прохорова Светлана)/
```

`#` не протекает дальше парсера — в архиве, выходных документах и стриминговом сервисе его нет.

Примеры входных файлов:
```
#История AWT   → title="История AWT", LEADERSHIP
#История       → title="История", LEADERSHIP
История Herbalife и истоки → title="История Herbalife и истоки", EDUCATIONAL
```

### 4. Формат имени файла (без изменений)

```
{YYYY.MM[.DD]} {type}[.{stream}]. {title} ({speaker}).ext
```

Для offsite день опускается (всегда `YYYY.MM`), используется только месяц.

## EventCategory

`EventCategory` enum (REGULAR/OFFSITE) **сохраняется** — он по-прежнему определяет логику формирования пути (разные правила для regular и offsite). Поле `event_category` в `VideoMetadata` остаётся.

## Изменение формата topic_folder

Для regular-мероприятий topic_folder меняется:
- **Было:** `{stream} {title} ({speaker})` — например `SV Контент (Пепелина Инга)`
- **Стало:** `{MM.DD} {stream}. {title} ({speaker})` — например `08.04 SV. Контент (Пепелина Инга)`

Изменения: дата перемещается из отдельной папки в начало topic_folder, разделитель между потоком и темой меняется с пробела на `. `.

## Миграция — детали

1. **Переименование папок:** Скрипт перестраивает 4-уровневую структуру в 3-уровневую
2. **`pipeline_results.json`:** Содержит сериализованный `archive_path` — скрипт обновляет поле `metadata.archivePath` в каждом файле
3. **`video_id`:** Не меняется (генерируется из даты+типа+потока+темы, не из пути)
4. **Rollback:** Скрипт создаёт JSON-маппинг old_path→new_path перед миграцией

## Ключевые решения

1. **Offsite — только месяц:** Выездные многодневные (3-5 дней), не более 5 в год, не бывает двух в одном месяце → день игнорируется
2. **«Выездные» убирается:** Лишний уровень группировки, дата в `event_group` даёт хронологию
3. **`#` маркер для историй:** Явный, однозначный, безопасен на всех ОС
4. **`EventCategory` сохраняется:** По-прежнему нужен для выбора логики формирования пути
5. **Фронтенд адаптируется:** Дерево строится динамически, но нужны правки параметров API-запросов
