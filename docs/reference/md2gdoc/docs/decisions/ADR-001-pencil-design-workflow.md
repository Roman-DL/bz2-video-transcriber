# ADR-001: Двухэтапный подход к разработке UI — Pencil + Code

**Статус:** Принято
**Дата:** 2026-02-23

## Контекст

В проекте bz2-video-transcriber UI генерировался навыком `frontend-design`, который создавал дизайн и код одновременно. Навык генерирует качественный, distinctive дизайн, но итерации по стилю и layout на уровне готового React-кода дорогие — каждое изменение требует правки кода, пересборки и визуальной проверки.

Пользователь (не дизайнер) хочет видеть и обсуждать визуальный дизайн **до** генерации кода, чтобы:
- Оценить общий стиль и направление
- Попросить изменить конкретные элементы
- Сравнить варианты
- Утвердить результат перед реализацией

## Решение

Используем двухэтапный подход через адаптированный навык `/frontend-design`:

```
Этап 1: Дизайн в Pencil
  Design thinking → Style guide → Визуальный макет (.pen) → Скриншот → Итерации с пользователем

Этап 2: Реализация в код
  Утверждённый макет → Анализ параметров из .pen → React + TypeScript + Tailwind код
```

### Инструменты

- **Pencil** (pencil.dev) — визуальный редактор через MCP-сервер в Claude Code. Работает с `.pen` файлами. Позволяет создавать макеты, итерировать, делать скриншоты.
- **Навык `/frontend-design`** — адаптирован для двухэтапного процесса. Содержит design thinking guidelines (типографика, цвета, composition) и правила генерации production-grade кода.

### Когда применять

| Ситуация | Подход |
|----------|--------|
| Новый экран / страница | `/frontend-design` (Pencil → код) |
| Значительная переработка UI | `/frontend-design` (Pencil → код) |
| Мелкая правка (текст, цвет, кнопка) | Напрямую в код |
| Баг в UI | Напрямую в код |

## Альтернативы

### 1. Только код (текущий подход в bz2-video-transcriber)
- (+) Быстрее для простых задач
- (-) Дорогие итерации по дизайну
- (-) Пользователь видит результат только после генерации кода

### 2. Pencil для дизайна + отдельный навык для кода
- (+) Чёткое разделение ответственности
- (-) Навык кодирования не знает дизайн-контекст
- (-) Два отдельных инструмента вместо одного workflow

### 3. Pencil для дизайна + кодогенерация Pencil
- (+) Единый инструмент
- (-) Pencil генерирует базовый HTML/CSS, не React-компоненты
- (-) Нет state management, hooks, API интеграции
- (-) Нет motion/animations

## Последствия

- `.pen` файлы макетов хранятся в `frontend/designs/` и коммитятся в репозиторий
- Навык `/frontend-design` — единая точка входа для UI задач
- При планировании фич с UI (в `/preflight`) — учитывать этап дизайна в Pencil
- Design thinking guidelines поддерживаются в навыке, не дублируются в docs
