# Процесс разработки

Полный цикл разработки с Claude Code — от требования до финализации.

---

## Обзор процесса

```
Requirement → Plan → /preflight → Реализация → /finalize → Релиз
```

Не все этапы обязательны — зависит от масштаба задачи.

---

## Типы задач и их процессы

### Крупная фича

```
1. docs/requirements/  → описать требование
2. docs/plans/         → создать план (Plan Mode)
3. /preflight          → проверить совместимость с архитектурой
4. Реализация
5. /finalize           → проверить документацию и drift
6. cz bump             → релиз (если значимая фича)
```

### UI экран или значительная переработка интерфейса

```
1. /frontend-design    → дизайн в Pencil (.pen)
2. Итерации макета с пользователем
3. Утвердили → генерация React + Tailwind кода
4. /finalize
```

Навык `/frontend-design` объединяет дизайн и реализацию. Подробности: `.claude/rules/frontend.md`, ADR-001.

### Быстрая доработка

```
1. Читать CLAUDE.md → реализация
2. Секция "После реализации" в CLAUDE.md — автоматическая проверка
```

### Баг

```
Простой (1-2 файла) → исправить напрямую
Сложный             → план → реализация
Архитектурный       → добавить ограничение в CLAUDE.md
```

## Артефакты разработки

### Requirements (`docs/requirements/`)

Описание **что** нужно сделать и **зачем**. Lifecycle:

```
Draft → In Progress → Done → Archive
```

- Создавать для значимых фич (не для баг-фиксов)
- Шаблон: `docs/requirements/REQUIREMENT-TEMPLATE.md`
- Индекс: `docs/requirements/README.md`

### Plans (`docs/plans/`)

Описание **как** реализовать требование. Lifecycle:

```
Draft → Active → Done → Archive
```

- Создавать для сложных задач (Plan Mode)
- Могут ссылаться на requirement
- Шаблон: `docs/plans/PLAN-TEMPLATE.md`
- Индекс: `docs/plans/README.md`

### ADR (`docs/decisions/`)

Фиксация **почему** выбран конкретный подход. Создавать когда:

- Выбор между несколькими подходами
- Новая абстракция или паттерн
- Интеграция внешнего сервиса
- Решение с долгосрочными последствиями

## Команды Claude Code

| Команда | Когда | Что делает |
|---------|-------|------------|
| `/preflight` | До реализации | Анализ совместимости с архитектурой |
| `/finalize` | После реализации | Проверка документации и drift detection |
| `/sync-docs` | Периодически | Полный аудит документации проекта |
| `/frontend-design` | Новый UI экран | Дизайн в Pencil → код |

Подробности: [COMMANDS-REFERENCE.md](COMMANDS-REFERENCE.md)

### Автоматическое поведение (без команд)

- **Rules** (`.claude/rules/`) загружают контекст модуля автоматически при работе с файлами
- **CLAUDE.md** секция "При планировании" — автоматическая проверка архитектуры
- **CLAUDE.md** секция "После реализации" — базовые проверки документации

Команды нужны для **глубокого анализа** сложных случаев.

---

## При планировании (Plan Mode)

Перед созданием плана изучи архитектуру:

- `docs/ARCHITECTURE.md` — как устроена система
- `docs/architecture/` — детали затрагиваемых подсистем
- `docs/decisions/` — почему приняты текущие решения
- `docs/requirements/` — активные требования
- `.claude/rules/` — правила модулей

---

## После реализации

Автоматическая проверка (секция в CLAUDE.md):

1. Архитектура изменилась? → обнови `docs/ARCHITECTURE.md`
2. Реализован модуль? → создай `docs/architecture/{module}.md`
3. Новое решение? → предложи ADR
4. Структура изменилась? → обнови CLAUDE.md
5. Rules актуальны? → обнови или создай
6. Edge case? → добавь ограничение в CLAUDE.md

Для глубокого аудита — `/finalize`. Для проверки всего проекта — `/sync-docs`.
