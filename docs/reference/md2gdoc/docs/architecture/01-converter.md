# 01 — Converter

Подсистема конвертации документов: MD → HTML → Google Doc (прямая) и Google Doc → HTML → MD (обратная).

**Обзор архитектуры:** [ARCHITECTURE.md](../ARCHITECTURE.md)

---

## Pipeline конвертации

### Прямая конвертация (MD → GDoc)

```
┌─────────┐    ┌──────────────┐    ┌──────────────┐    ┌─────────────────┐    ┌────────────┐
│ MD файл │───▶│ Parse        │───▶│ Convert      │───▶│ Image           │───▶│ Upload     │
│         │    │ frontmatter  │    │ MD → HTML    │    │ Resolution*     │    │ as GDoc    │
└─────────┘    └──────────────┘    └──────────────┘    └─────────────────┘    └────────────┘
                     │                    │                     │
                YAML metadata     Callouts → таблицы    * MVP: skip + warning
                strip / header    Таблицы, списки       Будущее: upload на GDrive
                                  Bold, italic, код
```

**Шаги:**

1. **Parse frontmatter** — извлечь YAML метаданные (`title`, `speaker`, `date`, `tags`). В зависимости от настройки: strip (убрать) или header (вставить в шапку документа)
2. **Convert MD → HTML** — Python-Markdown или mistune с расширениями для Obsidian callouts
3. **Image Resolution** — обработка ссылок на изображения (MVP: placeholder)
4. **Upload as GDoc** — загрузка HTML на Google Drive с автоконвертацией

### Обратная конвертация (GDoc → MD, для `two-way`)

```
┌───────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌─────────┐
│ Google Doc│───▶│ Export       │───▶│ Detect       │───▶│ Restore      │───▶│ MD файл │
│           │    │ as HTML      │    │ callouts     │    │ image links  │    │         │
└───────────┘    └──────────────┘    └──────────────┘    └──────────────┘    └─────────┘
                                           │                    │
                                     Таблица 2×1 +       Сопоставить с
                                     shading + emoji     file_mappings
                                     → > [!type]         Удалённые — убрать
```

**Шаги:**

1. **Export as HTML** — Google Drive API `Files.export` с mimeType `text/html`
2. **Detect callouts** — распознать каллауты по формату таблицы (см. ниже)
3. **Restore image links** — сопоставить изображения GDoc с оригинальными локальными ссылками
4. **Convert HTML → MD** — парсинг HTML обратно в Markdown

## Поддерживаемые элементы Markdown

| Элемент | MD синтаксис | HTML результат |
|---------|-------------|----------------|
| Заголовки | `# H1` — `###### H6` | `<h1>` — `<h6>` |
| Bold | `**text**` | `<b>text</b>` |
| Italic | `*text*` | `<i>text</i>` |
| Списки | `- item`, `1. item` | `<ul>/<ol>` + `<li>` |
| Таблицы | `\| col \|` | `<table>` |
| Ссылки | `[text](url)` | `<a href="url">` |
| Код (inline) | `` `code` `` | `<code>` |
| Код (block) | ` ``` ` | `<pre><code>` |
| Callouts | `> [!type] Title` | HTML таблица (см. ниже) |
| Изображения | `![alt](path)` | `<img>` / placeholder |

## Формат каллаутов в Google Doc

Каллауты конвертируются в детерминированный формат для обратного парсинга.

### Obsidian → HTML

```markdown
> [!warning] Важное предупреждение
> Текст каллаута с **форматированием** и [ссылками](url).
```

↓ конвертируется в:

```html
<table border="1" cellpadding="8" style="border-collapse: collapse; width: 100%;">
  <tr style="background-color: #FFF3CD;">
    <td><b>Важное предупреждение</b></td>
  </tr>
  <tr>
    <td>Текст каллаута с <b>форматированием</b> и <a href="url">ссылками</a>.</td>
  </tr>
</table>
```

### Правила обратного распознавания (GDoc → MD)

Таблица является каллаутом если **все три** условия выполнены:

1. Таблица **2 строки, 1 колонка**
2. Первая строка имеет **затенённый фон** (`background-color`)
3. Первая строка начинается с **иконки** (emoji)

### Маппинг типов каллаутов

| Иконка | Тип каллаута | Алиасы | Цвет фона |
|--------|-------------|--------|-----------|
| &#8505;&#65039; | `info` | `note` | `#D1ECF1` |
| &#128161; | `tip` | `hint` | `#D4EDDA` |
| &#9888;&#65039; | `warning` | `caution` | `#FFF3CD` |
| &#10060; | `danger` | `error` | `#F8D7DA` |
| &#128221; | `example` | — | `#E2E3E5` |
| &#10067; | `question` | `faq` | `#CCE5FF` |
| &#128204; | `abstract` | `summary` | `#E2E3E5` |
| &#128172; | `quote` | `cite` | `#E2E3E5` |

> Полный список типов и алиасов — в `topaz_nord_constants.js` (референсный проект). Цвета нейтральные.

## Формат изображений

### MD → GDoc (прямая)

| Формат в MD | Пример | Обработка |
|-------------|--------|-----------|
| Внешний URL | `![alt](https://...)` | Оставить `<img src="...">` — GDoc подтянет |
| Локальный путь | `![alt](./images/photo.png)` | Image Resolution: upload на GDrive → подставить URL |
| Obsidian wikilink | `![[photo.png]]` | Resolve путь → upload на GDrive → подставить URL |

**MVP:** локальные изображения → placeholder `[Изображение: photo.png]` + warning в логе. Внешние URL работают как есть.

**Будущая реализация:** `ImageResolver` загружает файлы на Google Drive (в подпапку `_images/` рядом с документом), получает publicUrl, подставляет в HTML.

### GDoc → MD (обратная)

Изображения в Google Doc **не добавляются и не меняются** пользователями. Могут быть **удалены**.

Стратегия:
1. Сравнить изображения в GDoc с известными из `file_mappings`
2. Оставшиеся — **сохранить оригинальные локальные ссылки** в MD
3. Удалённые из GDoc — **убрать ссылку из MD** (файл на диске не удалять)

## Обработка YAML Frontmatter

### Режим `strip` (убрать)

```markdown
---
title: "Название"
speaker: "Спикер"
---
# Содержимое
```

→ Frontmatter удаляется, в HTML попадает только содержимое.

### Режим `header` (шапка документа)

```markdown
---
title: "Название"
speaker: "Спикер"
date: "2026-02-23"
---
# Содержимое
```

→ Выбранные поля (настраиваемый список) вставляются в начало HTML как шапка:

```html
<p><b>Название</b></p>
<p>Спикер: Спикер | Дата: 2026-02-23</p>
<hr>
<!-- далее содержимое -->
```
