# md2gdoc — Спецификация проекта

> Документ требований для AI-разработки

**Версия:** 1.0
**Дата:** 2026-02-23
**Статус:** Draft

---

## Проблема

Для загрузки образовательных материалов (лонгридов) в базу знаний БЗ 2.0 требуется ссылка на Google документ. Сейчас конвертация из Markdown выполняется вручную: открыть MD в Obsidian → режим просмотра → скопировать → создать Google Doc → вставить → получить ссылку. Каждый документ — 5-10 минут ручной работы.

Предыдущая попытка автоматизации через n8n оказалась нестабильной и требовала платный CloudConvert для конвертации HTML → DOCX.

**Текущая ситуация:**
- Pipeline bz2-video-transcriber генерирует лонгриды в формате `.md`
- Для загрузки в БЗ нужен Google Doc со ссылкой
- Форматирование теряется при ручном копировании (каллауты, таблицы)

**Почему это проблема:**
- Ручной процесс не масштабируется при увеличении количества материалов
- Ошибки при копировании снижают качество документов
- Автоматизация через n8n не оправдала себя (нестабильность, стоимость CloudConvert)

---

## Решение

Самостоятельный веб-сервис с административной панелью, который мониторит папки (локальные и Google Drive) и автоматически конвертирует Markdown файлы в Google Docs.

### Ключевые идеи

1. **MD → HTML → Google Doc** — конвертируем MD в HTML, загружаем на Google Drive с автоконвертацией в Google Doc (бесплатно, без CloudConvert)
2. **Мониторинг папок** — отслеживание локальных директорий (watchdog) и Google Drive папок (polling) на появление новых/изменённых `.md` файлов
3. **Административная панель** — веб-интерфейс для настройки папок, просмотра логов, управления параметрами конвертации
4. **Универсальность** — работает с любыми MD файлами, не привязан к конкретному pipeline

### Архитектура

```
                    ┌──────────────────────────┐
                    │    Admin Panel (Web UI)  │
                    │  настройки, логи, статус │
                    └───────────┬──────────────┘
                                │ HTTP API
┌──────────────┐    ┌───────────┴──────────────┐    ┌──────────────────┐
│ Local Folder │───▶│      md2gdoc Service     │───▶│   Google Drive   │
│  (watchdog)  │    │                          │    │  (Google Doc)    │
├──────────────┤    │  1. Detect new .md       │    │                  │
│ Google Drive │───▶│  2. Parse MD → HTML      │    │  → Ссылка на     │
│  (polling)   │    │  3. Upload as Google Doc │    │    документ      │
└──────────────┘    │  4. Log result           │    └──────────────────┘
                    └──────────────────────────┘
```

## Пользователи и сценарии

### Первичный пользователь

Администратор базы знаний БЗ 2.0 — загружает образовательные материалы (лонгриды, саммари) и нуждается в ссылках на Google Docs.

### Основные сценарии

#### Сценарий 1: Первоначальная настройка

1. Открыть admin panel (https://md2gdoc.home)
2. Настроить Google OAuth credentials
3. Добавить watched folder (тип, путь, целевая папка, фильтр файлов)
4. Включить мониторинг

**Результат:** Сервис начинает отслеживать папку

#### Сценарий 2: Автоматическая конвертация

1. Pipeline bz2-video-transcriber сохраняет `longread.md` в `/archive/`
2. Сервис обнаруживает новый файл (watchdog)
3. Конвертирует MD → HTML → загружает как Google Doc
4. Записывает в лог: файл, ссылка, время

**Результат:** Google Doc доступен по ссылке без ручных действий

#### Сценарий 3: Ручная конвертация

1. Открыть admin panel
2. Загрузить MD файл (drag & drop) или указать путь
3. Выбрать целевую папку на Google Drive
4. Нажать "Конвертировать"

**Результат:** Ссылка на созданный Google Doc

## Scope

### Включено (MVP — v0.1)

- [ ] MD → HTML конвертер с поддержкой: заголовки H1-H6, bold, italic, списки, таблицы, ссылки, код
- [ ] Обработка Obsidian callouts (`> [!type]`) → HTML таблицы
- [ ] Обработка YAML frontmatter (strip / вставить в шапку документа, настраивается)
- [ ] Google Drive API: OAuth2 + upload HTML as Google Doc
- [ ] API endpoint: `POST /api/convert` (ручная конвертация)
- [ ] Базовый UI: форма загрузки файла + результат со ссылкой
- [ ] Деплой через Docker-compose (Traefik совместимый)

### Включено (v0.2 — мониторинг)

- [ ] Watchdog: мониторинг локальных папок
- [ ] Google Drive API: polling папок на изменения
- [ ] CRUD для watched folders (API + UI)
- [ ] Дедупликация по hash (не конвертировать один файл дважды)
- [ ] Фоновый worker для конвертаций

### Включено (v0.3 — admin panel)

- [ ] UI: список watched folders с статусами
- [ ] UI: таблица логов конвертаций
- [ ] UI: страница настроек (frontmatter, стили, polling)
- [ ] UI: Google OAuth flow через браузер

### Исключено (Out of Scope)

| Что отложено | Почему |
|--------------|--------|
| Изображения в MD | Ограничение HTML → Google Doc конвертации, лонгриды не содержат изображений |
| Embedded файлы, dataview | Специфика Obsidian, не используется в pipeline |
| Двусторонняя синхронизация (Google Doc → MD) | Значительно усложняет архитектуру, нет текущей потребности |
| Batch upload через UI | Мониторинг папок покрывает этот use case |
| Кастомные CSS темы | Нейтральные стили достаточны, можно добавить позже |

## Критерии успеха

### Минимальный успех

- [ ] Конвертация произвольного MD файла через web UI → получение ссылки на Google Doc
- [ ] Заголовки, списки, таблицы, bold/italic сохраняются в Google Doc
- [ ] Obsidian callouts конвертируются в читаемый формат
- [ ] YAML frontmatter не попадает в тело документа (или конвертируется в шапку)

### Полный успех

- [ ] Автоматический мониторинг папок: новый MD → Google Doc без ручных действий
- [ ] Admin panel: настройка папок, просмотр логов, управление Google OAuth
- [ ] Дедупликация: повторная конвертация только при изменении файла
- [ ] Стабильная работа 24/7 на сервере рядом с bz2-video-transcriber

## Технический контекст

### Предпочтительный стек

| Компонент | Технология | Почему |
|-----------|------------|--------|
| Backend | Python, FastAPI | Консистентность с bz2-video-transcriber, async support |
| Frontend | React (Vite) | Консистентность с bz2-video-transcriber |
| БД | SQLite | Файловая, без внешних зависимостей, достаточна для объёма |
| MD → HTML | Python-Markdown или mistune | Нативный Python, расширяемый (callouts, frontmatter) |
| File watcher | watchdog | Стандартная Python-библиотека для мониторинга FS |
| Google API | google-api-python-client, google-auth | Официальный SDK |
| Deploy | Docker, Traefik | Рядом с bz2-video-transcriber на том же сервере |

### Ограничения

| Параметр | Лимит | Обоснование |
|----------|-------|-------------|
| Размер MD файла | 10 МБ | Ограничение Google Drive API |
| Google Drive API | 300 запросов/мин | Rate limit Google API |
| Polling интервал | 1-60 мин (default 5) | Баланс скорости и ресурсов |
| Одновременных конвертаций | 3 | Не превышать rate limits |
| Длина имени документа | 255 символов | Ограничение Google Docs |

### Интеграции

- **Google Drive API** — Files.create с `mimeType: application/vnd.google-apps.document` для автоконвертации HTML → Google Doc
- **Google OAuth2** — Service Account (серверный, простой) или OAuth2 (гибкий, требует UI)
- **bz2-video-transcriber** — как источник MD файлов (через мониторинг папки `/archive/`)

## Глоссарий

| Термин | Определение |
|--------|-------------|
| Longread | Структурированный образовательный материал 5-10 страниц, генерируется pipeline bz2-video-transcriber |
| Callout | Блок-цитата Obsidian формата `> [!type] Title`, используется для выделения важной информации |
| Frontmatter | YAML-метаданные в начале MD файла (title, speaker, date, tags) |
| Watched folder | Папка (локальная или Google Drive), которую сервис мониторит на появление новых MD файлов |
| БЗ 2.0 | База знаний — внешняя система, куда загружаются материалы по ссылке на Google Doc |

## Открытые вопросы

- [ ] **Service Account vs OAuth2** — для серверного приложения Service Account проще (не нужен refresh token flow), но ограничен одним аккаунтом. OAuth2 гибче, но требует UI для авторизации
- [ ] **Обновление vs создание нового** — при изменении MD файла: обновлять тот же Google Doc (сохраняет ссылку) или создавать новую версию?
- [ ] **Поддержка изображений** — MD файлы из pipeline не содержат изображений, но универсальный инструмент может столкнуться с ними. Отложить или заложить сразу?
- [ ] **Стек frontend** — React (как в transcriber) или что-то более лёгкое (Svelte, plain HTML) для простого admin panel?

## Приложения

### Референсный проект (bz2-video-transcribe)

Исходные материалы из предыдущих попыток автоматизации:

| Файл | Что полезно |
|------|-------------|
| `docs/reference/Obsidian-Export/obsidian_to_gdocs.py` | Regex-паттерны каллаутов и тегов (Python, проверены на реальных документах) |
| `docs/reference/Obsidian → Google Docs/Obsidian → Google Docs.json` | n8n workflow — логика конвертации в Code нодах |
| `docs/reference/Obsidian → Google Docs/topaz_nord_constants.js` | 13 типов каллаутов + алиасы, helper-функции генерации HTML |

### Что берём из референсного проекта

| Компонент | Источник | Что берём |
|-----------|----------|-----------|
| Парсинг каллаутов | `obsidian_to_gdocs.py`, n8n Code нода | Regex для `> [!type]`, конвертация в HTML-блоки |
| Парсинг тегов | `topaz_nord_constants.js` | Regex для `#tag` → форматированный текст |
| Типы каллаутов | `topaz_nord_constants.js` | 13 типов + алиасы (note, info, tip, warning и др.) |
| Google Drive upload | n8n workflow (концепция) | Upload с `mimeType: application/vnd.google-apps.document` |

### Что НЕ берём

| Компонент | Причина |
|-----------|---------|
| n8n архитектура | Заменяется собственным сервисом |
| CloudConvert | Заменяется бесплатной автоконвертацией Google Drive |
| Topaz-Nord цвета | Устаревшая тема, используем нейтральные стили |
| Pandoc | Заменяется Python-библиотекой для MD → HTML |
| Hardcoded credentials | Заменяется конфигурацией через admin panel |
