# Промпт для новой беседы

---

## Анализ результатов двухпроходной очистки транскриптов

### Контекст

В v0.65 мы обновили систему очистки: глоссарий v4.0 (семантический формат + раздел people), промпт с 3 задачами, default модель claude-haiku-4-5, chunk_size 40K.

**Проблема:** задача 3 (причёсывание — удаление дублей, заиканий, техшума) полностью игнорируется Haiku на длинном тексте. Тестирование показало, что даже Sonnet не справляется с удалением дублей на 40K-чанках. Модели хорошо делают точечные замены (термины, имена), но не могут "выкинуть" целые фрагменты при проходе через длинный текст.

**Гипотеза:** если разделить задачи на два прохода (1-й — термины/имена, 2-й — очистка без глоссария), каждый будет выполнен качественнее. Без глоссария (~35K токенов) у модели значительно больше headroom для структурного анализа.

### Тест

Провели ручной тест в Claude Console (Haiku, температура 1) на фрагменте ~3500 символов из реального транскрипта (ФСТ Мужской клуб, Руцман Александр).

Файлы теста: `docs/reference/test-two-pass/`
- `fragment.md` — исходный фрагмент
- `pass1-glossary.md` — промпт прохода 1 (паразиты + термины/имена)
- `pass2-cleanup.md` — промпт прохода 2 (структурная очистка)
- `result 1.md` — результат прохода 1
- `result 2.md` — результат прохода 2

### Результаты

**Проход 1 (термины + имена): 15/15 — идеальный результат**

Все 12 ошибок имён исправлены, включая:
- "Анал Роэнс" → "Алан Лоренц"
- "Марк Джимрон" → "Марк Хьюз, Джим Рон" (разделил склеенные имена!)
- "Mark Crews" → "Стефан Грациани" (по контексту)
- "Клещьяна Рональда" → "Криштиану Роналду"
- "40 лет в Германии" → "40 лет в бизнесе" (восстановил смысл — Whisper исказил "в Herbalife")

Все термины (Herbalife ×6 вариантов, Супервайзер, Фаундер Серкл) и паразиты (ну, значит, короче) — исправлены/удалены.

**Проход 2 (очистка): хорошо работает**

Убрал переходные фразы ("Давай начнем", "Можно дальше", "Дальше, поехали"), презентационное ("Ролик потом", "можно следующую фотографию?"), диалоговый шум ("И до 46, сколько он там? - 46, 44."). Ни одно исправление из прохода 1 не было сломано.

### Что нужно обсудить

1. **Оценка результатов** — насколько значимо улучшение по сравнению с одним проходом? На коротком фрагменте (3.5K) даже один проход мог бы хорошо работать. Нужен ли тест на полном транскрипте (76K)?

2. **Стоит ли реализовывать?** — двухпроходная очистка стоит x2 от текущей. Haiku дёшев (оценка: ~$0.26/час → ~$0.52/час), но это всё же двойная стоимость. Оправдано ли улучшение качества?

3. **Как реализовать, если да** — два варианта:
   - Два последовательных вызова LLM в `TranscriptCleaner` (проход 1 с глоссарием, проход 2 без)
   - Или использовать текущий промпт для прохода 1 (задачи 1+2), а проход 2 — новый промпт (задача 3)

4. **Тест на полном транскрипте** — стоит ли перед реализацией прогнать полный 76K транскрипт через двухпроходную схему (через API или step endpoint)?

5. **Альтернативные варианты** — может быть, достаточно добавить ключевое правило в промпт ("НЕ удаляй повторы, НЕ причёсывай текст") и оставить один проход? Тест показал, что фокусировка модели на одной задаче работает лучше.

### Ограничения теста

- Фрагмент маленький (3.5K vs 40K в реальности) — на коротком тексте любая модель работает лучше
- Один запуск (температура 1) — нет статистики по стабильности
- Нет контрольного прогона того же фрагмента через текущий промпт (все 3 задачи вместе)
- Проход 2 тестирован на уже чистом тексте (мало что убирать) — для настоящего теста нужен фрагмент с дублями и заиканиями
