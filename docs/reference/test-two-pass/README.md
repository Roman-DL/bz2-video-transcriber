# Тест двухпроходной очистки

## Гипотеза

Haiku не может одновременно:
1. Заменять термины/имена по глоссарию (точечные замены)
2. Удалять структурный шум (дубли, заикания, техшум)

При обработке длинного текста модель тратит всё "внимание" на проход сквозь текст, и хватает ресурсов только на простейшие паттерн-замены.

**Предположение:** если разделить задачи на два прохода, каждый будет выполнен качественнее.

## Порядок теста

### Шаг 1: Проход 1 — терминология

1. Открыть Claude Console (Haiku)
2. System prompt: скопировать из `pass1-glossary.md` (секция "System prompt")
3. User prompt: скопировать из `pass1-glossary.md` (секция "User prompt"), вставив фрагмент из `fragment.md`
4. Сохранить результат

### Шаг 2: Проход 2 — очистка

1. Новый чат в Claude Console (Haiku)
2. System prompt: скопировать из `pass2-cleanup.md` (секция "System prompt")
3. User prompt: вставить результат прохода 1 в `<input>` тег
4. Сохранить результат

### Контрольный тест: один проход (текущий промпт)

Для сравнения — прогнать тот же фрагмент через текущий `config/prompts/cleaning/system.md` (с задачами 1+2+3 вместе).

## Файлы

- `fragment.md` — тестовый фрагмент (~3500 символов)
- `pass1-glossary.md` — промпт прохода 1 (термины + имена)
- `pass2-cleanup.md` — промпт прохода 2 (очистка шума)

## Ожидаемый вывод

Если двухпроходная очистка даёт заметно лучший результат по именам и терминам — стоит реализовать в коде (два последовательных вызова LLM в `TranscriptCleaner`).

Если разницы нет — фиксируем ограничение и оставляем один проход.
