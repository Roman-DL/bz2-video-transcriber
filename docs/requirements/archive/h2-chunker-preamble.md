# Пропуск преамбулы в H2 chunker

> Текст перед первым `##` заголовком не чанкируется — остаётся только в лонгриде для читателей

**Статус:** Ready
**Дата:** 2026-02-21
**Версии:** v0.74.x

---

## 1. Проблема

В `h2_chunker.py` баг в детекции преамбулы (текст перед первым `## `). Условие на строке 83 никогда не срабатывает — преамбула парсится как обычная секция с мусорным topic.

**Текущая ситуация:**
- Условие `not H2_PATTERN.search(f"## {section}")` всегда False (код сам подставляет `## `)
- Преамбула попадает в ветку `else` → topic = первая строка (весь абзац), content = остаток
- Создаётся мусорный чанк: topic — огромный абзац, word_count — минимальный

**Пример из прода** (лонгрид по теме "Спонсор, за которым идут"):

Преамбула лонгрида:
```
Меня зовут Светлана Дмитрук, я President's Team 15K 1D, в бизнесе Herbalife 13 лет...
Сегодня поговорим о том, что делает спонсора тем, за кем хотят идти...

## Рекрутировать можно только туда, во что ты сам веришь
```

Результат чанкирования — чанк #1:
- `topic`: весь первый абзац (~80 слов) — непригоден для поиска
- `word_count`: 41 — тонкий чанк, только второй абзац
- Содержимое: представление спикера + обзор темы — мета-информация, не предметные знания

---

## 2. Решение

**Пропускать преамбулу при чанкировании.** Без порогов, без очистки, без topic "Введение".

### Обоснование

Преамбула — это **входная точка для читателя**, не для поисковика:

- **Вариант "личная история"**: представление спикера, эмоциональное вступление. RAG-ценность нулевая — запросы по имени спикера покрываются `metadata.speaker`
- **Вариант "обзор темы"**: сжатый пересказ всех тем выступления. RAG-ценность есть, но дублирует поле `description` из `TranscriptChunks`, которое специально оптимизировано для поиска
- Преамбула **остаётся в лонгриде** (`longread.md`) — читатели видят вступление
- В чанки (`transcript_chunks.json`) попадают только тематические секции с H2 заголовками

### Принцип

```
Преамбула (текст до первого ##)  →  пропускается
## Первый H2 заголовок           →  чанк #1
## Второй H2 заголовок           →  чанк #2
...
```

## 3. Источники преамбулы

### Longread.to_markdown()

```markdown
---
type: "лонгрид"
video_id: "..."
---

# Название лекции

## Вступление
...
```

Преамбула: YAML frontmatter + H1. **Пропускаем** — первый чанк начнётся с `## Вступление`.

### Story.to_markdown()

```markdown
---
type: "leadership-story"
names: "..."
---

# История Успеха: Иванов А.

> [!abstract] Главный инсайт
> Текст инсайта...

---

## 1️⃣ Кто они
...
```

Преамбула: YAML frontmatter + H1 + Obsidian callout. **Пропускаем** — `main_insight` дублируется в `description`, первый чанк начнётся с `## Кто они`.

### Реальный LLM output (наблюдалось в проде)

```markdown
Сегодня поговорим о закрытии полной оценки — моменте, когда
технология проведена, а клиент стоит перед выбором...

## Закрытие полной оценки: как работать с возражениями
...
```

Преамбула: вводный текст от LLM. **Пропускаем** — обзорная информация покрыта `description`.

## 5. Тестирование

| Сценарий | Ожидаемый результат |
|----------|---------------------|
| YAML frontmatter + H1 + введение | Пропускается, чанки начинаются с первого H2 |
| Story: frontmatter + callout | Пропускается, чанки начинаются с `## Кто они` |
| LLM intro перед первым H2 | Пропускается |
| Нет преамбулы (markdown начинается с `## `) | Без изменений, как сейчас |
| Только преамбула, без H2 | 0 чанков (edge case) |
