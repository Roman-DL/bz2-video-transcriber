# Примеры системных промптов

> **Назначение:** Референс для создания системных промптов при реализации Фаз 3-4 pipeline.
>
> Эти промпты адаптированы для Claude Desktop (ручной режим). При автоматизации нужно объединить их с инструкциями и шаблонами.

---

## Лонгрид (educational)

```
Создай лонгрид обучающей темы по шаблону и инструкции в новом артефакте.

**Входные данные:**
- Транскрипт темы (во вложении)
- Инструкция и шаблон для лонгрида (во вложениях)
- Слайды презентации (во вложении) — текстовая выгрузка или фото

**Задача:**
1. Прочитай транскрипт полностью
2. Используй слайды для дополнения: точные формулировки, структура, данные которые спикер показал но не проговорил
3. Определи структуру темы (разделы по логике автора)
4. Перепиши тему от первого лица, убрав мусор транскрипции
5. Исправь ошибки терминов по глоссарию (в памяти проекта)
6. Убедись, что каждый раздел самодостаточен для RAG
7. Заполни метаданные

**Контекст:**
- Событие: [название]
- Дата: [дата]
- Спикер: [имя, статус]
```

**Связанные файлы:**
- [Educational_Longread_Instructions.md](Educational_Longread_Instructions.md)
- [Educational_Longread_Template.md](Educational_Longread_Template.md)

## Конспект (educational)

```
Теперь создай конспект по транскрипту, используя лонгрид как референс структуры.

**Входные данные:**
- Транскрипт (уже в контексте беседы)
- Лонгрид (уже в контексте — использовать как референс структуры)
- Инструкция и шаблон для конспекта (во вложениях)

**Задача:**
1. Определи тип темы: методическая / философская / смешанная
2. Выбери соответствующие блоки из шаблона
3. Извлеки ключевую информацию из транскрипта (инструменты, принципы, ошибки)
4. Выдели 3-5 ключевых цитат спикера (близко к оригиналу, с очисткой)
5. Сформулируй главный инсайт (1 предложение)
6. Заполни блок «Применение» конкретными действиями
7. Добавь ссылку на лонгрид в метаданных
8. Включи вопросы-ответы из зала (если были) — близко к оригиналу
```

**Связанные файлы:**
- [Educational_Summary_Instructions.md](Educational_Summary_Instructions.md)
- [Educational_Summary_Template.md](Educational_Summary_Template.md)

## Лидерская история (leadership)

```
Создай конспект лидерской истории по шаблону и инструкции.

**Входные данные:**
- Транскрипт истории (во вложении)
- Инструкция и шаблон для лидерской истории (во вложениях)
- Слайды (во вложении) — текстовая выгрузка или фото, если есть

**Задача:**
1. Прочитай транскрипт полностью
2. Если есть слайды — используй для дополнения дат, цифр, схем работы
3. Заполни релевантные блоки шаблона (пустые блоки удалить)
4. Исправь ошибки терминов по глоссарию (в памяти проекта)
5. Выдели 2-3 ключевые цитаты
6. Заполни метаданные (особенно speed, format, keywords)
7. Определи «Для кого эта история»
8. Сформулируй главный инсайт

**Контекст:**
- Событие: [название]
- Дата: [дата]
- Спикер: [имена, статус]
```

**Связанные файлы:**
- [Leadership_Story_Instructions.md](Leadership_Story_Instructions.md)
- [Leadership_Story_Template.md](Leadership_Story_Template.md)

## Адаптация для pipeline

При реализации в автоматическом pipeline эти промпты нужно адаптировать:

1. **Убрать ручные указания** — "во вложении", "в контексте беседы"
2. **Добавить метаданные программно** — событие, дата, спикер из ParseStage
3. **Убрать работу с глоссарием** — термины уже исправлены на этапе очистки
4. **Объединить в один промпт:**
   ```
   [Системный промпт]     — адаптированная версия запроса выше
   [Инструкции]           — из *_Instructions.md
   [Шаблон]               — из *_Template.md
   [Очищенный транскрипт] — результат CleanStage
   [Слайды]               — результат SlidesStage (если есть)
   ```
