# План: Упрощение очистки до глоссарной подстановки

## Контекст

Тестирование очистки на транскрипте 60K символов (ФСТ Дмитрук Светлана) выявило +3.3% увеличение текста вместо ожидаемого уменьшения. Анализ показал:

1. **Дублирование блока ~2900 символов** — overlap-зона (3000 символов) обработана моделью по-разному в двух чанках (разная капитализация «Независимым»/«независимым»), мерж не нашёл точное совпадение → весь overlap попал в результат дважды
2. **Слабое удаление слов-паразитов** на длинных текстах — известная особенность Haiku (ADR-014)
3. **Удаление паразитов не критично** — longread-модель (Sonnet) устраняет их при генерации

Решение: убрать задачу удаления паразитов, оставить только глоссарную подстановку, убрать overlap (не нужен для точечных замен).

## Изменения

### 1. Промпт `config/prompts/cleaning/system.md`

**Убрать:** задачу 1 (слова-паразиты), пример с паразитами, правило «85-98%», правило «НЕ удаляй повторы»

**Оставить:** задачу по терминологии (глоссарий), примеры глоссарных замен, правила сохранения авторской речи, формат ответа

**Добавить:** явное указание «выходной текст ≈ входному по длине, разница только в заменённых терминах»

**Переименовать роль:** «технический редактор» → «корректор терминологии»

### 2. `backend/app/services/cleaner.py`

**`_split_into_chunks()`** (строки 284-332):
- Убрать overlap-логику (строки 314-323): новый чанк начинается без overlap-предложений
- Параметр `self.chunk_overlap` больше не используется

**`_merge_chunks()`** (строки 350-397):
- Заменить на простую конкатенацию: `return " ".join(chunks)`
- Overlap-детекция больше не нужна

**`clean()` метод:**
- `num_predict` (строка 138): изменить с `max(len(chunk) * 0.5, 4096)` на `max(int(len(chunk) * 1.1 * 2.3), 4096)` — выходной текст ≈ входному, нужен запас ≈110%
- Валидация (строки 227-243): обновить пороги — reduction > 15% = warning (вместо 25%), > 25% = error (вместо 40%), т.к. при чистой подстановке текст не должен значительно уменьшаться
- Также добавить проверку на увеличение: если текст вырос > 5%, это warning (возможно модель добавила контент)
- Docstring класса (строки 28-42): обновить описание — убрать «Filler words removal»

### 3. `config/models.yaml`

- `chunk_overlap: 0` во всех context_profiles (large, medium, small) — или убрать параметр
- Комментарий к `chunk_size` в `large` профиле: пересчитать. Без удаления паразитов output ≈ input, значит `num_predict` выше → нужно больше output tokens → можно уменьшить chunk_size или оставить 40K

### 4. `docs/decisions/014-haiku-default-cleaning.md`

Добавить секцию **«Обновление v0.72: Фокус на глоссарной подстановке»**:

- Контекст: тестирование на 60K тексте показало дублирование из-за хрупкого overlap-мержа
- Анализ: удаление паразитов не влияет на качество лонгрида (Sonnet справляется), но создаёт проблемы (дублирование, перефразирование)
- Решение: оставить только глоссарную подстановку, убрать overlap
- Последствия: более надёжная очистка, устранение дублирования, фокус модели на одной задаче

### 5. `CLAUDE.md`

- Обновить описание pipeline в секции «Архитектура» (если нужно)

1. Запустить очистку того же транскрипта (`2026.02 ФСТ. Спонсор, за которым идут (Дмитрук Светлана).md`) — убедиться что дублирование исчезло

3. Размер очищенного текста ≈ оригиналу (±2-3%)
