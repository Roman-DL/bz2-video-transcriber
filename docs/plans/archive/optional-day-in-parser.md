# План: опциональный день в дате файла

## Контекст

Выездные мероприятия (offsite) проходят несколько раз в год, и пользователю неудобно указывать конкретный день — достаточно месяца. Регулярные еженедельные мероприятия по-прежнему указывают полную дату. Нужно сделать день опциональным в regex парсера, не создавая два отдельных парсера.

## Изменения

### 1. `backend/app/services/parser.py`

**Regex** (строка 45-52): сделать день опциональным:
```python
r'^(\d{4}\.\d{2}(?:\.\d{2})?)\s+'   # Date: 2025.04.07 or 2025.04
```

**Парсинг даты** (строка 243-246): два формата:
```python
try:
    if len(date_str) == 10:  # YYYY.MM.DD
        video_date = datetime.strptime(date_str, '%Y.%m.%d').date()
    else:  # YYYY.MM
        video_date = datetime.strptime(date_str, '%Y.%m').date()
        # day=1 по умолчанию от datetime
except ValueError as e:
    raise FilenameParseError(filename, f"Invalid date format: {e}")
```

`datetime.strptime("2026.02", "%Y.%m")` возвращает `datetime(2026, 2, 1)` — день автоматически = 1.

**Archive path** (строка 276-277): для regular с днём=1 — ничего не меняется, формат `MM.DD` работает. Пользователь всегда указывает день для regular, а для offsite день в пути и так не используется.

**Docstring** (строки 1-16): обновить формат и примеры.

### 2. Тесты (в `__main__` блоке parser.py)

Добавить 2 теста:
- **Offsite без дня**: `"2026.02 ФСТ. Спонсор, за которым идут (Дмитрук Светлана).mp4"` — парсится, `date.day == 1`
- **Offsite без дня (leadership)**: `"2026.02 ШБМ. История (Иванов Дмитрий).mp4"` — LEADERSHIP, day=1

### 3. Больше ничего не меняем

- `VideoMetadata.date` остаётся `date` — day=1 при отсутствии
- `saver.py` — не трогаем, `strftime` работает с day=1
- `generate_video_id` — не трогаем, `isoformat()` даст `2026-02-01`
- Формат ошибки `FilenameParseError` — обновить сообщение: `{date}` → `{YYYY.MM[.DD]}`
