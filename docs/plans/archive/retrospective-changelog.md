## План реализации

### 1. Записать ретроспективные записи в CHANGELOG.md

Добавить ниже существующей записи v0.80.0 (строка 8) все версии от v0.79.0 до v0.29.0 в формате Commitizen:

```
## X.Y.Z (YYYY-MM-DD)

### Feat

- описание на русском

### Fix

- описание на русском
```

**Ключевые решения:**

- **Все версии** из git log (v0.1–v0.79), не только из таблицы CLAUDE.md — итого ~65 записей
- **Sub-versions** (v0.55.1, v0.58.1, v0.73.1 и т.д.) сливаются в родительскую версию
- **Фиксы между версиями** — атрибутируются к следующей версии (как делает Commitizen)
- **Английские описания** (v0.29–v0.59) — переводятся на русский
- **Типы:** только `Feat`, `Fix`, `Refactor`, `Docs` (совместимо с парсером)
- **docs-only коммиты** (frontmatter, аудиты, миграции) — не включаются, кроме версий где docs = основной контент (v0.34, v0.47, v0.48, v0.55)
- **v0.1.0** — подробная запись: все Phases 0-13 + ранние фиксы как отдельные пункты Feat/Fix
- **Пропуски в нумерации** (v0.10–v0.11, v0.14–v0.19) — нормально, промежуточные коммиты атрибутируются в следующую явную версию

**Маппинг для английских коммитов:**

| Prefix | → Тип |
|--------|-------|
| Add, feat: | Feat |
| Fix, fix: | Fix |
| Improve, Redesign | Feat |
| Refactor, Separate, Unify | Refactor |
| Update documentation | Docs |

### 2. Удалить раздел «Текущий статус» из CLAUDE.md

Заменить строки 218–245 (раздел целиком) на:

```markdown
## Текущий статус

**Версия:** v0.81 • [Полная история изменений](CHANGELOG.md) (v0.1–v0.80)
```

**Обоснование:** Таблица дублировала CHANGELOG.md. Теперь CHANGELOG — единственный источник истории. AI при необходимости прочитает его напрямую. Секция «После реализации» в CLAUDE.md больше не требует обновления таблицы — `cz bump` обновит CHANGELOG автоматически.

### 3. Обновить инструкцию версионирования в CLAUDE.md

Строка 11 сейчас:
```
- **Версионирование:** После коммита новых фич — ОБЯЗАТЕЛЬНО предложи обновить версию в `frontend/package.json`
```

Заменить на:
```
- **Версионирование:** `cz bump` для релиза (обновляет VERSION, package.json, CHANGELOG.md, git tag). Предлагай при завершении значимых фич.
```

### 4. Добавить шаг «Рекомендация релиза» в /finalize

Добавить в `.claude/commands/finalize.md` новый шаг между Шагом 2 и Шагом 3 (перед проверкой размера CLAUDE.md):

```markdown
## Шаг 2.5: Рекомендация релиза

Проверь накопленные изменения с последнего тега:

1. Посмотри коммиты с последнего тега: `git log $(git describe --tags --abbrev=0)..HEAD --oneline`
2. Подсчитай feat/fix/refactor коммиты
3. Оцени нужен ли релиз:
   - **1–2 fix** → "Пока рано для релиза"
   - **feat или 3+ fix** → "Рекомендую релиз: `cz bump`. Сделать?"
4. Если пользователь согласен → выполни `cz bump --dry-run`, покажи результат, затем `cz bump`
```

Это реализует Сценарий D из requirements (§3).
