## Типы планов

| Тип | Когда | Источник |
|-----|-------|----------|
| **Phase** | Начало проекта | PROJECT-SPEC.md |
| **Feature** | Новая фича | requirements/*.md |
| **Refactoring** | Улучшение кода | Контекст проекта |
| **Bugfix** | Сложный баг | Issue или описание |

## Статусы

- **Draft** — обсуждение в Plan Mode
- **Approved** — план согласован
- **In Progress** — реализация в новой сессии
- **Done** → перемещается в `archive/`

## Как создать план

### Для новой фичи (из requirements/)

```
Изучи docs/requirements/{feature}.md и создай план реализации.
Сохрани в docs/plans/{feature}.md
```

### Для доработки (без requirements)

```
Спланируй реализацию: {описание задачи}
Сохрани план в docs/plans/{название}.md
```
