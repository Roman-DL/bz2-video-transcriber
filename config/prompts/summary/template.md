Верни JSON:

```json
{
  "essence": "2-3 абзаца: главная идея темы, какую проблему решает, для кого полезна",

  "key_concepts": [
    "Важное понятие или различие 1",
    "Важное понятие или различие 2",
    "Важное понятие или различие 3"
  ],

  "practical_tools": [
    "Инструмент/метод 1: краткое описание",
    "Инструмент/метод 2: краткое описание"
  ],

  "quotes": [
    "«Точная цитата спикера — яркая мысль»",
    "«Ещё одна важная цитата»"
  ],

  "insight": "Одно предложение — главная мысль, которую нужно унести",

  "actions": [
    "Конкретное действие для применения 1",
    "Конкретное действие для применения 2"
  ],

  "topic_area": ["продажи", "инструменты"],

  "tags": ["тег1", "тег2"],

  "access_level": "consultant"
}
```

## Правила заполнения

### essence
- 2-3 абзаца: главная идея + для кого полезна + какую проблему решает
- Писать от третьего лица (не от лица спикера)

### key_concepts
- Ключевые понятия, определения или различия из темы
- Каждый элемент — одно понятие с кратким пояснением

### practical_tools
- Инструменты или методы с описанием (для методической темы)
- Для философской темы — принципы вместо инструментов или пустой массив
- Формат: «Название: краткое описание»

### quotes
- Точные цитаты спикера в кавычках «»
- Выбирать яркие, запоминающиеся фразы
- Каждая цитата — завершённая мысль, понятная без контекста
- Цитаты должны быть дословными из транскрипта

### insight
- Одна мысль, которую человек унесёт с собой
- Одно-два предложения

### actions
- Конкретные действия для применения — столько, сколько реально даёт тема
- Приоритет: инструменты и упражнения, которые слушатель может начать использовать сразу
- Если спикер дал шаблон, чек-лист или упражнение — это действие
- Если нет чётких рекомендаций — пустой массив

### topic_area
- 1-3 области из списка: продажи | спонсорство | лидерство | мотивация | инструменты | маркетинг-план

### tags
- 3-5 тегов для поиска (названия инструментов, ключевые понятия)

### access_level
- consultant | leader | personal
- По умолчанию — consultant

ВАЖНО:
- Верни ТОЛЬКО валидный JSON
- Не добавляй поля, которых нет в шаблоне
- Пустые блоки — пустой массив [], НЕ удалять поле
- Цитаты должны быть дословными из транскрипта
- topic_area — массив строк из допустимого списка
