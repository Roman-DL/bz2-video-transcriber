# Инструкции для создания конспекта

## Принципы

1. **Структура важнее полноты** — выделить скелет темы, а не пересказать всё
2. **Конкретика** — цифры, названия, точные формулировки
3. **Цитаты для ключевых моментов** — прямые цитаты спикера (дословно!). Каждая цитата — завершённая мысль, понятная без контекста. Не дублировать одну идею несколькими цитатами
4. **Не придумывать** — если данных нет, пропустить блок
5. **Выбирать блоки по типу темы** — не все блоки обязательны
6. **Исправлять ошибки транскрибации** — использовать глоссарий

---

## Типы тем и выбор блоков

Определи тип темы по содержанию и заполни соответствующие блоки.

### Методическая тема
Спикер даёт конкретные инструменты и пошаговые инструкции.

Акцент на блоках:
- **practical_tools** — названные инструменты (дом-магазин, 3D-чат, солнышко) с описанием
- **actions** — чёткая последовательность шагов для внедрения
- **key_concepts** — определения терминов и ключевые различия

### Философская тема
Спикер говорит о мышлении, принципах, мотивации.

Акцент на блоках:
- **key_concepts** — фундаментальные идеи, убеждения, принципы
- **insight** — главная мысль для запоминания
- **quotes** — яркие цитаты, передающие суть философии
- **practical_tools** — можно заполнить принципами вместо инструментов

### Смешанная тема
Спикер сочетает конкретные инструменты с философией, мотивацией или историями.

Акцент на блоках:
- **key_concepts** — и определения, и принципы/убеждения
- **practical_tools** — конкретные инструменты с описанием
- **actions** — включить и инструменты для внедрения, и упражнения из темы
- **quotes** — баланс: яркая мысль + конкретный антипример/совет

## Правила извлечения

### Что включать
- Конкретные инструменты с названиями
- Пошаговые алгоритмы (если есть)
- Принципы и убеждения спикера
- Примеры с цифрами и деталями
- Названные формулы и фреймворки спикера (если спикер явно даёт формулу, модель, схему — это всегда key_concept)
- Интерактивные упражнения с аудиторией (фокусная группа, мозговой штурм, заполнение шаблона — это всегда practical_tool + action)
- Яркие цитаты (дословно из транскрипта!)
- Предостережения и антипримеры

### Что НЕ включать
- Общие фразы без конкретики
- Повторы одной мысли
- Отступления от темы
- Технические моменты (аплодисменты, паузы)
- Слова ведущих до/после темы

### Работа с неполными данными
- Инструмент упомянут, но не объяснён → указать название + «детали не раскрыты»
- Нет чёткого алгоритма → пропустить блок actions или оставить пустой массив
- Нет примеров → пустой массив
- Нет Q&A → не добавлять вопросы-ответы
- Цитата неразборчива → не включать

## Классификация

### Допустимые значения topic_area

| Значение | Описание |
|----------|----------|
| `продажи` | Работа с клиентами, первички, продажи программ |
| `спонсорство` | Работа с командой, планирование, 3D-чаты |
| `лидерство` | Построение организации, первое поколение |
| `мотивация` | Философия, мышление, преодоление кризисов |
| `инструменты` | Конкретные методики (дом-магазин, солнышко) |
| `маркетинг-план` | Статусы, квалификации, роялти |

Выбери 1-3 области, наиболее точно отражающие содержание.

### Допустимые значения access_level

| Значение | Кто видит |
|----------|-----------|
| `consultant` | Все консультанты (по умолчанию) |
| `leader` | Лидеры уровня СТ+ |
| `personal` | Только для личного пользования |

По умолчанию используй `consultant`.

## Мультиспикерный контент

> **Применяй ТОЛЬКО если в задании есть блок «Мультиспикерный контент».** Без него — игнорируй эту секцию полностью.

### Со-спикеры
- Цитаты с атрибуцией: `> «Цитата» — Беркин А.`
- Ключевые мысли привязывать к конкретному спикеру

### Линейка выступлений
- Подразделы по участникам: `### Фамилия Имя`
- Каждый подраздел — key_concepts и quotes этого участника

### Q&A
- Блок «Вопросы и ответы» — если были вопросы из зала
- Формат: вопрос + ответ, без SpeakerN
