Верни JSON:

```json
{
  "essence": "2-3 абзаца: главная идея темы, какую проблему решает, для кого полезна",

  "key_concepts": [
    "Важное понятие или различие 1",
    "Важное понятие или различие 2",
    "Важное понятие или различие 3"
  ],

  "practical_tools": [
    "Инструмент/метод 1: краткое описание",
    "Инструмент/метод 2: краткое описание"
  ],

  "quotes": [
    "«Точная цитата спикера — яркая мысль»",
    "«Ещё одна важная цитата»"
  ],

  "insight": "Одно предложение — главная мысль, которую нужно унести",

  "actions": [
    "Конкретное действие для применения 1",
    "Конкретное действие для применения 2"
  ],

  "topic_area": ["продажи", "инструменты"],

  "tags": ["тег1", "тег2"],

  "access_level": "consultant"
}
```

## Правила заполнения

- **essence**: 2-3 абзаца. Главная идея + для кого полезна + какую проблему решает
- **key_concepts**: 3-5 ключевых понятий/различий из темы
- **practical_tools**: 2-4 инструмента/метода (если тема методическая). Если философская — пустой массив или принципы
- **quotes**: 2-4 точные цитаты в кавычках. Выбирать яркие, запоминающиеся
- **insight**: Одна мысль которую человек унесёт с собой
- **actions**: 2-3 конкретных действия для внедрения
- **topic_area**: 1-3 области из списка: продажи | спонсорство | лидерство | мотивация | инструменты | маркетинг-план
- **tags**: 2-5 тегов для поиска (названия инструментов, ключевые понятия)
- **access_level**: consultant | leader | personal

ВАЖНО:
- Верни ТОЛЬКО валидный JSON
- Не добавляй блоки, которых нет в теме
- Цитаты должны быть дословными из транскрипта
- topic_area — массив строк из допустимого списка
