## Принципы

1. **Вступление** — контекст темы, почему она важна, что будет рассмотрено
2. **Заключение** — краткий итог, главная мысль, призыв к действию (если был)
3. **Голос автора** — пиши от первого лица, как будто спикер подводит итог

## Формат ответа

Верни JSON:

```json
{
  "introduction": "1-2 абзаца вступления. От первого лица если спикер представлялся, или от третьего если это краткий контекст.",
  "conclusion": "1-2 абзаца заключения. Если спикер подводил итог — сохрани его. Если нет — кратко резюмируй главную мысль.",
  "section": "Один из: Обучение, Продукты, Бизнес, Мотивация",
  "subsection": "Подраздел внутри секции",
  "tags": ["тег1", "тег2", "тег3"],
  "access_level": 1
}
```

## Правила классификации

### Секции:
- **Обучение** — тренинги, развитие навыков, методики работы
- **Продукты** — информация о продуктах, применение, результаты
- **Бизнес** — построение бизнеса, продажи, рекрутинг
- **Мотивация** — истории успеха, вдохновение, личностный рост

### Уровни доступа:
- **1** — для всех консультантов (базовая информация)
- **2** — для спонсоров (продвинутые темы)
- **3** — для TAB Team (управленческие темы)
- **4** — для администраторов (внутренняя информация)

ВАЖНО:
- Верни ТОЛЬКО валидный JSON
- Если заключения в теме не было — напиши краткое резюме (2-3 предложения)
- Теги: 3-5 релевантных, существительные в именительном падеже
