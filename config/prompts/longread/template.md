# Шаблон: Лонгрид обучающей темы

## Структура выходного JSON

```json
{
  "introduction": "Вступление (1-2 абзаца): контекст темы, почему важна",
  "sections": [
    {
      "title": "Название раздела по содержанию",
      "content": "Изложение от первого лица..."
    }
  ],
  "conclusion": "Заключение (если спикер подводил итог)",
  "topic_area": ["область1"],
  "tags": ["тег1", "тег2", "тег3"],
  "access_level": "consultant"
}
```

## Правила заполнения

### introduction
- 1-2 абзаца
- Контекст темы: почему важна
- Если спикер представляется — включить кратко

### sections
- Разделы по логике автора (не менее 3)
- Заголовки по содержанию (не «Раздел 1»)
- Каждый раздел самодостаточен для RAG
- Изложение от первого лица
- Для выступления 30-90 минут оптимально 8-15 разделов — не дроби подтему на отдельные разделы без необходимости

### conclusion
- Итог, если спикер подводил
- Может быть пустым, если не было явного заключения

### topic_area
- 1-3 области из списка: продажи, спонсорство, лидерство, мотивация, инструменты, маркетинг-план
- Выбирать по основному содержанию

### tags
- 3-7 тегов для поиска
- Конкретные термины, инструменты, концепции

### access_level
- `consultant` — базовые знания, доступно всем
- `leader` — продвинутые техники для лидеров
- `personal` — личный опыт, ограниченный доступ
