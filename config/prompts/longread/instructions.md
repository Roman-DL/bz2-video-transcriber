# Инструкции: Создание лонгрида обучающей темы

## Принципы

1. **Голос автора** — писать от первого лица, сохранять характерные обороты
2. **Логика изложения** — следовать структуре, которую выбрал спикер
3. **Чистота текста** — убрать повторы, оговорки, ошибки транскрибации
4. **Полнота** — сохранить все значимые мысли, примеры, истории
5. **Читаемость** — разбить на разделы с заголовками
6. **RAG-ready** — каждый раздел должен быть самодостаточным

---

## Что убирать

### Технический мусор
- Слова-паразиты: «ну», «вот», «как бы», «короче»
- Незаконченные фразы и самоперебивания
- Повторы одной мысли разными словами
- Ошибки транскрибации
- Реплики ведущих, аплодисменты, паузы

### Отступления
- Не относящиеся к теме истории
- Организационные объявления
- Приветствия и благодарности (кроме значимых)

---

## Что сохранять

### Обязательно
- Все ключевые мысли и тезисы
- Примеры и истории (с деталями)
- Цифры, названия, конкретику
- Объяснения «почему» и «зачем»
- Предостережения и антипримеры
- Вопросы из зала и ответы

### Авторский стиль
- Характерные выражения спикера
- Риторические вопросы
- Эмоциональные акценты
- Обращения к аудитории (адаптировать к читателю)

---

## Работа со слайдами

Если в данных есть текст со слайдов — используй его для обогащения лонгрида.

### Что извлекать из слайдов
- **Цифры и статистику** — если спикер сказал «смотрите на слайд» без озвучивания
- **Структуру темы** — заголовки слайдов могут стать заголовками разделов
- **Схемы** — описать текстом или списком
- **Термины** — точные названия инструментов
- **Цитаты** — если на слайде есть текст цитаты

### Как интегрировать
- **Естественное включение** — вплести данные в текст как будто спикер их озвучил
- **Описание схемы** — если на слайде таблица/диаграмма, описать структурой (список, таблица)
- **Ссылка на визуал** — «На схеме показано...» — только когда нет способа передать словами

### Чего НЕ делать со слайдами
- Не описывать дизайн слайдов (цвета, шрифты)
- Не включать информацию, не относящуюся к теме
- Не дублировать — если спикер уже проговорил данные, слайд не нужен

## Правила редактирования

### Преобразование устной речи в письменную

**Было (транскрипт):**
> И вот, значит, я смотрю на этот дом-магазин, ну, и понимаю, что там, ну, как бы, проблема.

**Стало (лонгрид):**
> Смотрю на дом-магазин и сразу понимаю проблему.

### Обращения к аудитории

**Было:**
> Кто согласен, да? Поднимите руки.

**Стало:**
> (убрать — не относится к содержанию)

**Было:**
> Вы сейчас, когда меня будете слушать, конспектируйте вопросы.

**Стало:**
> (убрать — не относится к содержанию)

### Сохранение живости

**Было:**
> Я такая: подожди, а у нас же целая вселенная продуктов!

**Стало:**
> Думаю: подожди, у нас же целая вселенная продуктов!

## Разметка для RAG

Каждый раздел должен быть **самодостаточным** — понятным без предыдущих разделов.

### Хорошо:
```
## Дом-магазин

Дом-магазин — это универсальная таблица, я её называю
«открытая книга бизнеса». Сюда записывается досье на
каждого клиента...
```

### Плохо:
```
## Второй инструмент

Теперь про него. Это тоже важная таблица...
```

Если раздел ссылается на предыдущий — добавить контекст:
```
## Солнышко

Помимо дом-магазина, есть второй инструмент для
планирования — солнышко. Это визуальная схема
клиентской группы...
```

## Классификация

### Допустимые значения topic_area

| Значение | Описание |
|----------|----------|
| `продажи` | Работа с клиентами, первички, продажи программ |
| `спонсорство` | Работа с командой, планирование, 3D-чаты |
| `лидерство` | Построение организации, первое поколение |
| `мотивация` | Философия, мышление, преодоление кризисов |
| `инструменты` | Конкретные методики (дом-магазин, солнышко) |
| `маркетинг-план` | Статусы, квалификации, роялти |

Выбери 1-3 области по основному содержанию.

### Допустимые значения access_level

| Значение | Кто видит |
|----------|-----------|
| `consultant` | Все консультанты (по умолчанию) |
| `leader` | Лидеры уровня СТ+ |
| `personal` | Только для личного пользования |

По умолчанию используй `consultant`.

## Чек-лист качества

- Ошибки транскрибации исправлены
- Убраны повторы и слова-паразиты
- Сохранена логика изложения автора
- Каждый раздел самодостаточен (для RAG)
- Примеры и истории сохранены с деталями
- Данные со слайдов интегрированы (если были)
- Вопросы-ответы включены (если были)
- Текст читается плавно, без спотыканий
- Уровень детализации одинаков от начала до конца — не «сворачивай» текст к финалу
